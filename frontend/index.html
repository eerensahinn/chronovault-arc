Time-locked messages · Arc Testnet

Write today.
Deliver it whenever time says so.

ChronoVault locks a message on-chain until a date you choose. No one — not even you — can read it early. The contract itself enforces the wait.

See how it works ↓
Process

Three steps, enforced by code

There's no admin key that can open a capsule early — the unlock condition lives in the contract, not in a promise.

01

Write

Compose a message and pick the date it should become readable — a day, a month, a year from now.

02

Lock

Sign one transaction. The message is committed to Arc Testnet and sealed until that timestamp is reached.

03

Reveal

Once the unlock time passes, the contract releases the message to anyone who asks — publicly and permanently.

About the network

Built on Arc

Arc is an open, EVM-compatible Layer-1 blockchain developed by Circle — the company behind USDC. It's designed as financial infrastructure: predictable, dollar-denominated fees, sub-second transaction finality, and native support for stablecoins as gas.

ChronoVault is an independent, community-built project running on Arc's public testnet. It isn't an official Circle or Arc product — just one small example of what you can build on the network.

Want to build your own? Start at docs.arc.network.
